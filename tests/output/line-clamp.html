Line Clamp Values

These cards cover the legacy `-webkit-line-clamp` flow, the modern unprefixed `line-clamp` property, the `no-ellipsis` modifier, and an unclamped control. Browsers that do not support the modern values will simply render those cards without truncation.

Legacy One-Line Clamp
A single-line clamp should still append an ellipsis when the copy runs far beyond the width of the card and the browser clips the extra content.
display:-webkit-box
-webkit-line-clamp: 1
Legacy Two-Line Clamp
This two-line example mirrors the pattern used by feed cards, search results, and changelog teasers where the last visible fragment needs to pick up the ellipsis automatically.
-webkit-line-clamp: 2
No explicit text-overflow declaration.
Legacy Three-Line Clamp
Longer editorial text often wants one more line of headroom before the truncation kicks in, especially in tablet cards or responsive dashboard summaries.
-webkit-line-clamp: 3
Modern Two-Line Clamp
The modern shorthand expresses the same behavior in one declaration, which makes it easier to combine with container queries and utility-driven component systems.
line-clamp: 2
Modern No-Ellipsis
Some components need the visible lines clipped cleanly without appending an ellipsis glyph, for example when another affordance already signals that more content exists.
line-clamp: 2 no-ellipsis
Unclamped Control
This card uses the same copy as the others but leaves the content fully visible, making it easier to compare the truncated and untruncated layouts side by side in the exported output.
line-clamp: none
